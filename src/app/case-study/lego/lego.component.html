Lego CSS Library

Objective

Massive style libraries can block creativity, consistency, and contain more code than is necessary for a project. When I was first learning to program I used Twitter Bootstrap and Material Design. The downside was a considerable amount of code that I would never use. I would spend countless hours rewriting specific styles to my liking. When I noticed how much time I was spending doing this on every project LegoCSS was born.

Challenge

All of it. I had in-depth knowledge of CSS and SASS but designing and building an entire library, thinking out all of the edge-cases was a larger project than I imagined.

Learned

I began with a static HTML document that served as my library. It only existed on my computer and would take what I needed. I not only advanced my SASS skills but also learned more about Angular 2+. The entire documentation for this library uses Angular 6 because I wanted to become more familiar with it.

Other things I learned in the process:

  • Angular Routing
  • Creating npm scripts
  • Accessibility Standards
  • Optimization and SEO
